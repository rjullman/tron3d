User input

Control your own bike using the arrow keys/'awd'. Also an interactive menu with game options.

Computer control over some elements of the scene

Computer driven bikes during single player mode. The computer controlled bikes are programmed to drive away from upcoming collisions at right angles if possible, though they are also programmed to occasionally (randomly) make turns to add some sort of error / interesting paths.

View frustum culling

Only draws the objects in the scene that are inside the view frustum of the camera. This optimization along with level of detail control greatly increased the framerate.

Level of detail control

The bikes lay pipes behind them that add to the scene complexity. The detail of these pipes is decreased as the player is further away from them. This optimization along with view frustum culling greatly improved framerate.

Collision detection

Check each bike in the scene for collisions with the scene or any of the previously laid trails. The method for collisions with the scene is a simple check of whether the bike goes inside the objects. The method for collisions with the trails is a check of whether the segment connecing each trail path intersects with the segment composed of the bike front and the immediate next position it will take. Additional checks had to be made to deal with special cases like parallel lines.
